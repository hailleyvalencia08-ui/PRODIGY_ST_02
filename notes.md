# Notes: Compatibility Testing Approach

## 🧭 Methodology
- Manual exploratory testing across multiple browsers and devices.
- Focused on layout, navigation, functionality, and responsiveness.
- Used Chrome DevTools for device emulation and real devices for validation.

## 🔍 Thought Process
- Started with homepage and navigation to ensure core functionality.
- Checked cart and checkout as critical e‑commerce flows.
- Verified responsiveness by resizing browser windows and testing on mobile.
- Logged defects with screenshots for evidence.

## ⚠️ Challenges
- Safari mobile had inconsistent cart updates.
- Firefox search bar required button click instead of Enter key.
- Footer links not properly mapped.
